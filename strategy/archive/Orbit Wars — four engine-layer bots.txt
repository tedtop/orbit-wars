Orbit Wars — four engine-layer bots + "learn from the top bots without their code"

 Context

 The top public leaderboard is one shared engine — Slawek Biel's orbit_lite ("The Producer") —
 run with different config presets (agents/opponents/ORBIT_LITE_FAMILY.md). Beating the LB is a
 policy-layer problem on top of a known engine, not a new heuristic from scratch. comet_reaper
 (agents/comet_reaper/main.py) is our validated baseline = that engine (producer-v2, #1 in 2P & 4P)
 - never-crash guard + terminal-phase surge + comet-evac.

 The four highest-EV improvements each attack a structural weakness the entire family shares:
 opponents modeled as do-nothing, shallow 4P play, greedy/no-search, and a blunt global roi_threshold.
 Separately, the user asked how to learn from the private top-10 teams when we can't see their code and
 naive behavior-cloning can't exceed its teacher.

 Part A — Four bots

 Each bot = a fork of agents/comet_reaper/ (its main.py + vendored orbit_lite/) into
 agents/<name>/, changing exactly one thing, gated behind a knob whose zero value reproduces
 comet_reaper — so every bot is a clean A/B against the comet_reaper control and the new behaviour is
 tunable. Bots: precog, kingmaker, helmsman, maestro.

 1. precog — opponent-response modeling.
 - Problem: score_candidates (orbit_lite/planner_core.py:89) scores my waves against
 garrison_status, a projection where opponents do nothing (only reactivity = reinforce_size_beta
 floor, comet_reaper/main.py:76). ORBIT_LITE_FAMILY.md calls this the #1 shared exploit surface.
 - Change: before scoring, predict each opponent's likely launches this turn and apply them into the
 projection. v1 proxy (cheap, recommended): for each opponent reuse build_target_shortlist
 (planner_core.py:271) + safe_drain (planner_core.py:587) from their perspective (player_id=opp)
 = "send the safe surplus at their best target"; apply via apply_private_planned_launches (already
 used at main.py:355). Score my candidates against the post-opponent-move board.
 - Knob: opp_model_strength (0 ⇒ comet_reaper). Escalation v2: run the full planner per opponent
 (true 1-ply) if the proxy pays — watch per-turn wall-clock.

 2. kingmaker — 4P coalition / meta-play.
 - Problem: competitive_score (planner_core.py:75-86) sums opponents equal-weight
 (opp = net.sum(-1) − me, line 85). v2-gru's additive ffa_* bonuses were ≈noise and comet_reaper
 dropped them, so go deeper than bonuses.
 - Change: weighted opponent term — overweight the current leader (prefer plays that hurt whoever's
 winning), discount opponents already bleeding each other, plus a "don't provoke while leading" term.
 Active only when player_count >= 4. Knobs: leader_weight, etc.

 3. helmsman — shallow lookahead / search.
 - Problem: _greedy_select is myopic per-turn; no lookahead over our own future turns.
 - Change: beam search over the top-M wave-sets, roll each forward a few turns with orbit_lite's own
 movement/garrison projection, score the resulting state, pick the best line. Hard wall-clock cap +
 graceful fallback to greedy (Kaggle timeout = instant loss). Knobs: search_beam, search_depth,
 time_budget_ms.

 4. maestro — dynamic config controller (the v2-gru slot, done right).
 - Problem: v2-gru built a GRU to modulate roi_threshold/max_waves/ffa per turn but shipped it
 disabled (USE_GRU_CONTROLLER=True but GRU_WEIGHTS_AVAILABLE=False) — runs on a hand heuristic.
 comet_reaper only has a fixed terminal-phase surge.
 - Change: port v2-gru's extract_global_features (24-dim) + apply_strategy_controller scaffold and
 ship a better controller — improved heuristic, or actually-trained 4-output weights (self-play/BC).
 Reuse TinyGRUStrategyController, _clamp_config_values from the v2-gru notebook.

 Validation (all four): gym round-robin vs the comet_reaper control, 2P & 4P, ≥40 games / seed sweep,
 report win% / avg placement and per-turn time. Reuse the gym check pattern (rl/rl_agent.py:_check,
 make("orbit_wars")), the OpenSkill arena (arena.py), and bench.py. A bot is a submission candidate
 only if win% > 50 vs the control and it stays within the per-turn time budget.

 Part B — Learn from the top bots WITHOUT their code (idea 5)

 Framing correction: we can't see their code, but we CAN see their behavior —
 (obs, action) per step for every prize-zone team via pipeline/pull_topbot_episodes.py +
 extract_moves.py --min-rating 1500. BC/IRL/offline-RL need behavior, not source. Naive BC on
 prize-zone data hasn't even been run yet (bc_train.py is validated only on local low-rating data).
 The "reverse-engineering that failed" (reverse-engineering-.../replay_analysis.py) is a behavioral
 profiler (reaction time, phase volume, comet rate), not a policy learner.

 Naive BC caps at the teacher; ranked ways to exceed them from their logs:

 - Track A (recommended) — Objective recovery / preference ranking. Each move is a revealed
 preference: it beat the alternatives they passed. For each replay state, regenerate orbit_lite's
 candidate waves, compute per-candidate features, and fit weights so the team's actual move ranks top
 (pairwise logistic/ranking loss — tractable, data-efficient). Plug the recovered weights into a
 parameterized competitive_score, then let our deeper planner (precog/helmsman) optimize that
 objective better than they do. Answers "what are they optimizing for" — what reverse-engineering
 wanted but couldn't read off code. New: rl/objective_recovery.py.
 - Track B — Engine-as-teacher + tuning. The whole public family is orbit_lite presets (we own the
 code). CMA-ES/Optuna the config in the gym to beat the public presets → that tuned engine is a teacher
 ≥ public bots → BC (bc_train.py) → self-play past it (selfplay.py). Teacher is internal & rising,
 not external & fixed. New: pipeline/tune_config.py.
 - Track C — Mistake-mining. Run prize-zone replay states through orbit_lite's exact combat-flow
 projector; flag dominated moves (declined +EV captures, do-nothing-blind-spot losses); BC on "their
 mistake → the correction." Beats them where they're weak. New: pipeline/mine_mistakes.py.
 - Track D — Opponent-conditioning. Repurpose the fingerprints (replay_analysis.py) as a runtime
 archetype classifier → switch counter-config. Feeds precog + maestro.
 - Track E (heavier, later) — Offline RL (IQL/CQL/AWAC) on (s,a,s',terminal-reward): textbook
 "exceed the logging policy from fixed logs," but data-hungry/finicky — stage after A–C.
 - Track F — Self-play league with the public bots / clones as fixed benchmark opponents
 (selfplay.py already started): don't imitate, just guarantee we beat them.

 Step 0: pull prize-zone episodes + run the existing BC on them to see where naive imitation lands —
 the baseline every track above is trying to beat.

 Verification

 - Each bot: gym check (rl/rl_agent.py:_check pattern) + N-game round-robin vs comet_reaper (2P & 4P)
 via arena.py/bench.py; require win% > 50 vs control and per-turn time within budget.
 - Idea-5: Track A — held-out top-bot move-prediction top-1 accuracy vs the BC baseline, then
 recovered-weights engine vs stock comet_reaper in the gym. Track B — tuned config beats public presets
 in the gym.

 Decisions & execution order

 Resolved: bot #1 is named precog; objective-recovery (Track A) leads — it's the highest-EV
 use of the prize-zone data and it upgrades the scorer that both precog and kingmaker build on.

 Sequenced execution:
 1. Step 0 + Track A (lead). pull_topbot_episodes.py → extract_moves.py --min-rating 1500; run
 existing BC on prize-zone data as the imitation baseline; then build rl/objective_recovery.py —
 regenerate orbit_lite candidate waves per replay state, fit per-candidate scoring weights by pairwise
 ranking so each team's actual move ranks top. Output: a parameterized competitive_score + recovered
 weight vector. Gate: held-out top-1 move-prediction accuracy beats the BC baseline.
 2. precog (bot #1) — opponent-response modeling, built on the parameterized scorer from step 1.
 3. kingmaker (bot #2) — 4P weighted-opponent term, also consuming the recovered scorer.
 4. helmsman (bot #3) — shallow search; maestro (bot #4) — dynamic config controller.
 5. In parallel as compute allows: Track B (engine-as-teacher tuning), Track C (mistake-mining),
 Track D (opponent-conditioning) feeding precog/maestro.

 Each bot is gym-validated vs the comet_reaper control (2P & 4P, win% > 50, per-turn time within budget)
 before it becomes a submission candidate.
